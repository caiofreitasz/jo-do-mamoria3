Nesta atividade, você irá criar uma avaliação de múltipla escolha onde as perguntas e alternativas são geradas dinamicamente com JavaScript. O objetivo é aplicar os seguintes conceitos estudados: Estruturação da lista de perguntas e alternativas no JavaScript usando objetos. Criação da função mostraPergunta() para inserir o enunciado da pergunta no HTML (DOM). Estruturação das alternativas como objetos { texto, correta }. Criação da função mostraAlternativas() para gerar botões de escolha (radio buttons) dinamicamente. Manipulação do DOM para inserir e organizar os elementos da avaliação na tela. Você recebeu um arquivo base com: Uma área para CSS que pode personalizar (cores, fontes, espaçamento). Estrutura mínima de HTML para exibir título, perguntas e botão de envio. Código JavaScript já comentado, que cria a lógica de exibição das perguntas e alternativas. O que você deve fazer Personalizar o array perguntas no JavaScript, criando pelo menos 5 perguntas de múltipla escolha. Garantir que cada alternativa seja um objeto com o texto e a indicação se é correta ou não. Alterar o estilo visual na área de CSS para deixar a avaliação mais agradável. Personalizar a função enviarAvaliacao() para exibir uma mensagem final diferente ou até mostrar a pontuação. Inserir no topo da página o nome completo do aluno e número de chamada de forma visível. Não apagar os comentários do código, mas pode adicionar mais explicações. Publicação e entrega Publique seu projeto no GitHub Pages. No Google Classroom, envie: O link do GitHub Pages (para abrir e testar a atividade). O link do repositório no GitHub com o código. Regras O trabalho deve estar em um único arquivo HTML (HTML + CSS + JS juntos). O código deve estar funcionando e gerar todas as perguntas na tela quando a página for aberta. O botão “Enviar” deve funcionar, mostrando ao menos uma mensagem de confirmação. A estrutura de objetos e as funções devem ser mantidas conforme o exemplo (pode renomear variáveis, mas não remover as funções principais). Critérios de avaliação (1,0 ponto) 0,4 – Estrutura correta das perguntas e alternativas como objetos. 0,3 – Implementação funcional de mostraPergunta() e mostraAlternativas(). 0,2 – Manipulação correta do DOM para exibir conteúdo dinamicamente. 0,1 – Personalização visual, inclusão do nome + número de chamada, e clareza do código com comentários.
